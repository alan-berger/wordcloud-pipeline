# wordcloud_pipeline

An automated pipeline that polls a remote URL for new incrementally-numbered text files, generates a [wordcloud](https://github.com/amueller/word_cloud) image from each one, and optionally patches metadata into an HTML/PHP page and sends a push notification via [ntfy](https://ntfy.sh).

Originally built for the [Security Now](https://www.grc.com/securitynow.htm) podcast by Steve Gibson, but designed to work with any regularly-published, incrementally-numbered text source.

The wordcloud on [alanberger.me.uk](https://alanberger.me.uk) is generated by this pipeline.

---

## Features

- Polls a configurable URL pattern for new text files — stops cleanly when the next one isn't available yet
- Catches up automatically if a run is missed (processes multiple new files in one pass)
- Generates a per-file archive image and updates a canonical "latest" image
- Extracts title and date metadata from the text file using configurable regex patterns
- Optionally patches an HTML/PHP page with updated episode metadata
- Rolling backups of the patched HTML file (configurable count)
- Optional push notifications via ntfy when a new file is processed
- All configuration in a single `config.ini` — no Python knowledge required to set up
- Designed to run as a cron job

## Requirements

- Python 3.9+
- `wordcloud` Python package

```bash
pip install wordcloud
```

On a cPanel shared host, use a virtualenv:

```bash
# Create a Python app in cPanel > Setup Python App, then:
source /home/username/virtualenv/wordcloud_env/3.9/bin/activate

```

## Installation

1. Clone or download this repository into a directory on your server:

```bash
git clone https://github.com/alan-berger/wordcloud_pipeline.git
cd wordcloud_pipeline
```

2. Copy the example config and edit it:

```bash
cp config.ini.example config.ini
```

3. Edit `config.ini` with your settings (see [Configuration](#configuration) below).

4. Place your assets (mask image, stopwords file) in the `assets/` directory, or update the paths in `config.ini` to point to wherever you keep them.

5. Test the pipeline manually:

```bash
python wordcloud_pipeline.py
```

On a virtualenv:

```bash
/home/username/virtualenv/wordcloud_env/3.9/bin/python wordcloud_pipeline.py
```

6. Once happy, add a cron job (see [Cron Setup](#cron-setup)).

## Configuration

All configuration lives in `config.ini`. Copy `config.ini.example` as your starting point — it contains a comment for every option.

### `[pipeline]`

| Key | Description | Example |
|---|---|---|
| `url_template` | URL pattern for the text files. Use `{n}` as the incrementing number placeholder. | `https://www.grc.com/sn/sn-{n}.txt` |
| `seed_number` | The last number you have already processed. The pipeline starts from `seed_number + 1`. Ignored once `state.txt` exists. | `1069` |
| `output_dir` | Directory where images are written. | `/home/username/public_html/images` |
| `latest_filename` | Filename for the always-current image, overwritten on every successful run. | `wordcloud-latest.png` |
| `archive_filename` | Filename pattern for per-number archive images. Use `{n}` as placeholder. | `wordcloud-{n}.png` |

### `[wordcloud]`

| Key | Description | Default |
|---|---|---|
| `width` | Image width in pixels. | `800` |
| `height` | Image height in pixels. | `800` |
| `background` | Background colour (CSS name or hex). | `white` |
| `mask` | Path to a mask image. The wordcloud will be shaped to the non-white areas of the image. Leave blank for a rectangular cloud. | _(blank)_ |
| `stopwords` | Path to a plain text stopwords file (one word per line). Leave blank to use the built-in English stopwords. | _(blank)_ |
| `cli_path` | Path to the `wordcloud_cli` binary. Leave blank to auto-detect. | _(blank)_ |

### `[metadata]`

These settings control how the pipeline extracts a title and date from each text file. Both use Python regex with a named capture group called `value`.

| Key | Description | Default (matches Security Now format) |
|---|---|---|
| `title_pattern` | Regex to extract a title. Must contain `(?P<value>...)`. Leave blank to skip. | `^TITLE:\s+(?P<value>.+)$` |
| `date_pattern` | Regex to extract a date. Must contain `(?P<value>...)`. Leave blank to skip. | `^DATE:\s+(?P<value>.+)$` |
| `date_format` | Python `strptime` format string for parsing the extracted date. | `%B %d, %Y` |

For Security Now transcripts, each file begins with a header block:

```
DATE:     March 10, 2026
TITLE:    You can't hide from LLMs
```

The default patterns extract these values directly. Adjust them to match whatever structure your own text files use.

### `[html_patch]`

Optionally patches a metadata block (episode number, title, date, image cache-buster) into an HTML or PHP file after each successful run.

| Key | Description | Default |
|---|---|---|
| `enabled` | Set to `true` to enable. | `false` |
| `file` | Absolute path to the file to patch. | _(blank)_ |
| `backup_count` | Number of rolling backups to keep (named `.bak1` through `.bakN`). | `3` |

> **Important:** The regex used for patching is written for a specific HTML structure and lives in the `patch_html_file()` function in `wordcloud_pipeline.py`. If you want to use this feature for your own site, you will need to adapt that function to match your own page layout. The comments in the source code explain the group structure.

**Backup behaviour:** Before every write, the existing file is copied to `.bak1`. Previous backups are shifted: `.bak1` → `.bak2` → `.bak3`, and `.bak3` (the oldest) is dropped. To recover a previous version:

```bash
cp /path/to/index.php.bak1 /path/to/index.php
```

### `[ntfy]`

Send a push notification after each successfully processed item. The notification title includes the item number and title (if available); the body contains the full pipeline log for that run.

| Key | Description | Default |
|---|---|---|
| `enabled` | Set to `true` to enable. | `false` |
| `server` | ntfy server URL. | `https://ntfy.sh` |
| `topic` | Your ntfy topic name. | _(blank)_ |

Install the ntfy app on your phone or desktop: [ntfy.sh](https://ntfy.sh). If using the public `ntfy.sh` server, choose a topic name that is unique and difficult to guess, as all topics are public by default.

## Cron Setup

Run the pipeline twice daily (adjust the Python path to your virtualenv):

```
0 6,18 * * * /home/username/virtualenv/wordcloud_env/3.9/bin/python /home/username/wordcloud_pipeline/wordcloud_pipeline.py >> /home/username/wordcloud_pipeline/pipeline.log 2>&1
```

To run every 4 hours instead:

```
0 */4 * * * /home/username/virtualenv/wordcloud_env/3.9/bin/python /home/username/wordcloud_pipeline/wordcloud_pipeline.py >> /home/username/wordcloud_pipeline/pipeline.log 2>&1
```

The pipeline is safe to run frequently — it exits cleanly with "Nothing to do" if no new files are available.

---

## State file

After the first successful run, a `state.txt` file is created in the same directory as the script. It contains the number of the last successfully processed item. Delete it to reset the pipeline, or edit it to re-process from a specific number.

## Security Now example

This is the original use case the pipeline was built for. Security Now publishes a weekly transcript at a predictable URL:

```
https://www.grc.com/sn/sn-{n}.txt
```

Each transcript begins with a metadata header:

```
EPISODE:  #1069
DATE:     March 10, 2026
TITLE:    You can't hide from LLMs
```

The pipeline detects when a new transcript is available (often before it appears on the website listing), generates a wordcloud shaped to a circular mask, and updates the homepage.

A minimal `config.ini` for Security Now looks like this:

```ini
[pipeline]
url_template    = https://www.grc.com/sn/sn-{n}.txt
seed_number     = 1069
output_dir      = /home/username/public_html/images
latest_filename = securitynow-wordcloud.png
archive_filename = wc-{n}.png

[wordcloud]
width      = 800
height     = 800
background = white
mask       = ./assets/circle800.jpg
stopwords  = ./assets/stop_words_english.txt

[metadata]
title_pattern = ^TITLE:\s+(?P<value>.+)$
date_pattern  = ^DATE:\s+(?P<value>.+)$
date_format   = %B %d, %Y
```

## Assets

Place your mask image and stopwords file in an `assets/` subdirectory, or configure any paths you prefer in `config.ini`.

A black-filled circle mask image (`circle800.jpg`, 800×800px) and an English stopwords list are the assets used in the Security Now example. You can substitute any mask image — the wordcloud will fill the non-white pixels of the image.
